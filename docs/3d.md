# 🧠 Arquitetura Gráfica – Projeto Bare Metal (RPi3 + VC4)

## 🎯 Objetivo

Construir uma stack gráfica própria em **Rust**, com:

- Backend específico para VideoCore IV (RPi3).
- Arquitetura portável para outros hardwares com UEFI.
- Desacoplamento entre API gráfica e hardware.
- Controle total sobre o pipeline de renderização.

---

# 🧱 Princípios Fundamentais

### 1. UEFI é apenas trampolim

O EDK2/UEFI será usado apenas para:
- Boot e descoberta de memória.
- Framebuffer inicial (essencial para debug).

> ⚠️ **ATENÇÃO:** **NÃO** será usado como stack gráfica nem para aceleração 3D.

### 2. Separação de Responsabilidades

A arquitetura é dividida em camadas para garantir modularidade:

1. **UEFI / EDK** (Bootloader)
2. **Kernel / HAL** (Abstração de hardware)
3. **Backend GPU** (VC4 hoje)
4. **Mini Mesa** (Camada intermediária)
5. **API Gráfica** (GL-like)

---

# 🧩 Componentes da Arquitetura

## 📦 1. Platform Layer (HAL)

Responsável por MMIO, alocação DMA, barreiras de memória e tradução de endereços CPU → GPU (*bus address*).

```rust
pub trait PlatformGpuSupport {
    fn mmio_map(&self, base: usize, size: usize) -> *mut u8;
    fn alloc_dma(&mut self, size: usize, align: usize) -> Result<DmaBlock, Error>;
    fn cpu_to_gpu_addr(&self, phys: usize) -> u32;
}
```

## 🎮 2. Backend GPU

Implementação específica do hardware. No caso do VC4, cuida das control lists, configuração de binning e sincronização.

```rust
pub trait GpuBackend {
    fn init(&mut self) -> Result<(), Error>;
    fn submit(&mut self, cmds: &[GpuCommand]) -> Result<Fence, Error>;
    fn wait_idle(&mut self) -> Result<(), Error>;
}
```

## 🧠 3. Mini Mesa (Core)

Rastreia o estado, valida comandos e gerencia a tradução para a IR (Representação Interna).

```rust
pub enum GpuCommand {
    ClearColor { rgba8: u32 },
    BindVertexBuffer { addr: u32, stride: u16 },
    BindPipeline { id: u32 },
    Draw { first: u32, count: u32 },
}
```

## 🧪 4. API Gráfica (GL-like)

Interface pública inspirada em OpenGL para facilitar o uso.

```rust
pub trait GraphicsApi {
    fn clear(&mut self, color: [f32; 4]);
    fn draw_triangle(&mut self, vertices: &[Vertex]);
    fn present(&mut self);
}
```

# 🛠️ Estratégia e Roadmap

## 🧪 Backend de Software (Fallback)

O foco aqui é validar o pipeline e facilitar o debug sem depender da GPU real. Se o triângulo renderizar corretamente via CPU, o erro no hardware estará no backend.

## 🚧 Roadmap Técnico

- **Fase 1:** Framebuffer básico e HAL estável.
- **Fase 2:** Software Renderer funcional integrado à IR.
- **Fase 3:** Backend VC4 mínimo (Clear screen).
- **Fase 4:** Pipeline completo (Binning e Draw).
- **Fase 5:** Expansão (Depth buffer, Texturas e Shaders).

---

# ⚠️ Insights e Riscos

- **Bus Address:** Nunca use endereços físicos da CPU diretamente na GPU.
- **Tile-based Rendering:** O VC4 trabalha com Vertex → Binning → Tiles → Rendering.
- **Serialização:** A GPU não entende structs de Rust; ela consome streams binários de bytes.
